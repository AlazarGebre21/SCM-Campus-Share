# Software Configuration Management Plan (SCMP)

## 1. Introduction
This document describes the Software Configuration Management (SCM) plan for the SCM Campus Share project.

## 2. Purpose
The purpose of this SCMP is to define the configuration management activities, responsibilities, and procedures for the project.

## 3. Scope
This plan applies to all software components, documentation, and related artifacts of the SCM Campus Share system.

## 4. Configuration Management Activities
- Configuration Identification
- Configuration Control
- Configuration Status Accounting
- Configuration Audits

## 5. Roles and Responsibilities
- Configuration Manager
- Development Team
- Quality Assurance Team

## 6. Tools and Infrastructure
- Version Control System: Git
- Repository: GitHub
- Issue Tracking: GitHub Issues

## 7. Configuration Control Board (CCB)
The CCB is responsible for reviewing and approving change requests.

## 8. Baseline Management
All baselines will be documented in the Baseline-Records directory.
